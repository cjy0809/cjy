# 社区老年活动中心管理系统 - 架构文档

## 项目概述

本项目是一个面向社区老年活动中心的综合管理系统，采用前后端分离架构。系统为老年用户提供活动报名、场地预约、健康记录、社区服务等功能，同时为管理员和工作人员提供数据分析和系统管理功能。

### 技术栈

**后端技术栈：**
- Spring Boot 2.7.14
- Spring Security（JWT认证）
- MyBatis Plus 3.5.3.1
- MySQL 8.0.33
- Redis（缓存）
- WebSocket（实时通信）
- 支付宝SDK（支付集成）
- Apache POI（Excel导出）
- Swagger/OpenAPI（API文档）

**前端技术栈：**
- Vue 3
- Vite（构建工具）
- Element Plus（UI组件库）
- Pinia（状态管理）
- Vue Router（路由管理）
- Axios（HTTP请求）
- ECharts（数据可视化）

#### 2.1.2 功能流程

**登录流程：**
1. 前端调用 `POST /api/user/login`，提交手机号和密码
2. UserController 接收请求，调用 UserService.login()
3. UserServiceImpl 验证用户信息，使用 BCrypt 加密验证密码
4. 验证成功后，使用 JwtUtil 生成 JWT token
5. 返回 UserLoginResponse，包含 token 和用户基本信息
6. 前端将 token 存储到 localStorage，后续请求携带 token

**权限控制：**
- 管理员（ADMIN）：所有权限
- 工作人员（STAFF）：查看和编辑老年用户，管理活动、场地、服务
- 老年用户（ELDERLY）：只能查看和修改自己的信息，参与活动、预约场地

#### 3.1.2 路由守卫

```javascript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  const isAuthenticated = !!userStore.token
  const userRole = userStore.userRole

  // 检查是否需要登录认证
  if (to.meta.requiresAuth) {
    if (!isAuthenticated) {
      next('/login')
      return
    }

    // 检查角色权限
    if (to.meta.role) {
      let hasPermission = false
      if (Array.isArray(to.meta.role)) {
        hasPermission = to.meta.role.includes(userRole)
      } else {
        hasPermission = userRole === to.meta.role
      }

      if (!hasPermission) {
        ElMessage.error('没有权限访问此页面')
        next(from.path)
        return
      }
    }
  }

  next()
})
```

```javascript
export const useUserStore = defineStore('user', () => {
  const token = ref(localStorage.getItem('token') || '')
  const userInfo = ref(JSON.parse(localStorage.getItem('userInfo') || '{}'))
  
  const isLoggedIn = computed(() => !!token.value)
  const userRole = computed(() => userInfo.value?.role || '')
  const isAdmin = computed(() => userRole.value === 'ADMIN')
  const isStaff = computed(() => userRole.value === 'STAFF')
  const isElderly = computed(() => userRole.value === 'ELDERLY')

  const userLogin = async (phone, password) => {
    const response = await login(phone, password)
    if (response.code === 200) {
      token.value = response.data.token
      userInfo.value = {
        id: response.data.userId,
        username: response.data.username,
        name: response.data.name,
        role: response.data.role,
        avatar: response.data.avatar
      }
      localStorage.setItem('token', token.value)
      localStorage.setItem('userInfo', JSON.stringify(userInfo.value))
      return { success: true }
    }
    return { success: false }
  }

  const userLogout = async () => {
    await logout()
    token.value = ''
    userInfo.value = {}
    localStorage.removeItem('token')
    localStorage.removeItem('userInfo')
  }

  return {
    token,
    userInfo,
    isLoggedIn,
    userRole,
    isAdmin,
    isStaff,
    isElderly,
    userLogin,
    userLogout
  }
})
```

```javascript
import axios from 'axios'
import { ElMessage } from 'element-plus'
import { useUserStore } from '@/stores/user'

const request = axios.create({
  baseURL: '/api',
  timeout: 30000
})

// 请求拦截器
request.interceptors.request.use(
  config => {
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers['Authorization'] = `Bearer ${userStore.token}`
    }
    return config
  },
  error => {
    return Promise.reject(error)
  }
)

// 响应拦截器
request.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code !== 200) {
      ElMessage.error(res.message || '请求失败')
      if (res.code === 401) {
        const userStore = useUserStore()
        userStore.userLogout()
        window.location.href = '/login'
      }
      return Promise.reject(new Error(res.message || '请求失败'))
    }
    return res
  },
  error => {
    ElMessage.error(error.message || '网络错误')
    return Promise.reject(error)
  }
)

export default request
```

## 六、部署说明

### 6.1 后端部署

1. **环境要求**
   - JDK 17+
   - Maven 3.6+
   - MySQL 8.0+
   - Redis 6.0+

2. **配置文件**
   - 修改 `application.yml` 中的数据库连接信息
   - 配置支付宝相关参数
   - 设置文件上传路径

3. **启动命令**
   ```bash
   cd backend
   mvn clean package
   java -jar target/center-1.0.0.jar
   ```

### 6.2 前端部署

1. **环境要求**
   - Node.js 16+
   - npm 或 yarn

2. **配置文件**
   - 修改 `vite.config.js` 中的代理配置
   - 设置后端API地址

3. **构建命令**
   ```bash
   cd frontend
   npm install
   npm run build
   ```

4. **部署**
   - 将 `dist` 目录部署到Nginx或其他Web服务器
   - 配置反向代理到后端API
